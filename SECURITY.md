# Security Policy

## Supported Versions

The following versions of VishKron are currently supported with security updates.

| Version        | Supported |
| -------------- | --------- |
| Latest release | ✅ Yes     |
| Older releases | ❌ No      |

Only the most recent stable version of VishKron receives security updates.
Users are strongly encouraged to upgrade to the latest version.

---

## Reporting a Vulnerability

If you discover a security vulnerability in VishKron, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, report it privately using one of the following methods:

• Open a private security advisory on GitHub
• Contact the maintainer via the official project repository

Please include the following information:

* A clear description of the vulnerability
* Steps to reproduce the issue
* A proof-of-concept or example (if possible)
* The version of VishKron affected

---

## Responsible Disclosure

We ask that you follow responsible disclosure practices:

* Do not publicly disclose the vulnerability before it is fixed.
* Allow reasonable time for the maintainer to investigate and release a patch.
* Work with the maintainer if clarification or testing is required.

---

## Response Timeline

Because VishKron is maintained by a single developer, response times may vary.

Typical targets:

* Initial response: within **3–7 days**
* Fix or mitigation: **as soon as reasonably possible**

---

## Scope

This security policy applies only to the official VishKron source code repository.

The following are **out of scope**:

* Third-party integrations
* Modified forks of VishKron
* Local misconfiguration by users
* Data loss due to improper backups or user actions

---

## Security Considerations

VishKron is designed as a **self-hosted, offline-first tool**.

Users are responsible for:

* Securing their own systems
* Protecting stored data
* Performing regular backups

The maintainer does not provide guarantees regarding data protection or regulatory compliance.

---

## Acknowledgements

Responsible security researchers who report valid vulnerabilities may be acknowledged in the project's release notes unless they prefer to remain anonymous.
